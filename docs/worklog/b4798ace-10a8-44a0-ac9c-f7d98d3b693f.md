- 워크플로가 서버에서 다루는 서비스명이 **`phoenix-dashboard`, `phoenix-eval-worker`** (로컬 compose의 `dashboard`/`eval-worker`와 다름) → 서버의 `~/servers/docker-compose.yml`에는 이 이름으로 정의돼 있어야 함

## 🤖 Claude

네, 못 가져옵니다. GitHub Secrets는 **write-only** 설계예요.

- 등록한 시점 이후로는 UI/API/CLI 어디에서도 평문 값을 다시 읽을 수 없음
- `gh secret list`는 이름·업데이트 시각만 보여줌 (값 X)
- Actions 로그에서도 secret 값이 출력되면 자동으로 `***`로 마스킹됨
- workflow 안에서 `echo $SECRET`을 해도 마스킹되어 노출 안 됨

따라서 값을 모르겠으면 방법은 하나뿐입니다 — **새 값으로 덮어쓰기** (`gh secret set NAME`).

만약 `APP_ENV` 내용을 다시 확인하고 싶다면, 워크플로가 마지막으로 그걸 어디에 썼는지 봐야 해요 — `~/servers/phoenix/.env`에 풀어놓으니까 배포 서버에 SSH로 들어가서 `cat ~/servers/phoenix/.env` 하면 현재 값을 볼 수 있습니다 (서버에 접근 가능하다면).

## 🤖 Claude

GitHub Secret에 저장된 private key는 못 가져옵니다 (방금 얘기한 것처럼 write-only). 그래서 **새 키페어를 발급해서 양쪽을 갱신**하는 게 정답입니다.

순서는 이렇습니다:

**1. 로컬에서 새 키페어 생성**
```bash
ssh-keygen -t ed25519 -f ~/.ssh/phoenix_deploy -C "github-actions-deploy" -N ""
```
→ `~/.ssh/phoenix_deploy` (private), `~/.ssh/phoenix_deploy.pub` (public)

**2. 서버의 `~/.ssh/authorized_keys`에 public key 추가**

서버 접근 방법에 따라 달라집니다:
- 서버에 직접 로그인 가능 (콘솔/물리/다른 키): `cat ~/.ssh/phoenix_deploy.pub | ssh user@host 'cat >> ~/.ssh/authorized_keys'`
- 패스워드 로그인 살아있으면: `ssh-copy-id -i ~/.ssh/phoenix_deploy.pub user@host`
- 서버가 본인 PC라 직접 키보드/모니터 접근 가능: `~/.ssh/authorized_keys`에 public key 한 줄 추가

**3. GitHub Secret 갱신**
```bash
gh secret set PC_SSH_KEY < ~/.ssh/phoenix_deploy
```
(또는 GitHub UI에서 Settings → Secrets → PC_SSH_KEY → Update)

**4. 동작 확인**
```bash
ssh -i ~/.ssh/phoenix_deploy user@host 'echo ok'
```
잘 되면 push해서 workflow가 통과하는지 보면 됩니다.

질문: 지금 그 PC 서버에 접근할 다른 방법(직접 키보드, 다른 SSH 키, 패스워드 로그인)이 있나요? 거기에 따라 2번 단계가 갈립니다.

## 🤖 Claude

tmux에서 윈도우 관련 종료 명령:

| 목적 | 명령 |
|---|---|
| 현재 윈도우만 종료 | `Ctrl-b &` (확인 후 y) 또는 `tmux kill-window` |
| 현재 세션의 **모든 윈도우** 종료 (= 세션 종료) | `tmux kill-session` |
| 현재 윈도우 빼고 나머지 다 종료 | `Ctrl-b :` → `kill-window -a` |
| tmux 서버 전체 종료 (모든 세션·윈도우 한 번에) | `tmux kill-server` |

"전체 종료"가 현재 세션의 윈도우 모두를 말한 거면 → `tmux kill-session`
tmux 자체를 다 끝내고 싶은 거면 → `tmux kill-server`

## 🤖 Claude

몇 가지 가능성이 있어요:

**1. tmux 안에 있는지 확인**
```bash
echo $TMUX
```
빈 값이면 tmux 안이 아니에요 (그냥 셸). 그러면 Ctrl-b는 셸 단축키(한 글자 뒤로)로 동작합니다.

**2. prefix 키가 다른 걸로 바뀐 경우**
`~/.tmux.conf`에서 `set -g prefix C-a` 같은 설정이 있으면 Ctrl-a 등으로 바뀝니다. 확인:
```bash
grep -i prefix ~/.tmux.conf 2>/dev/null
tmux show-options -g prefix
```

**3. prefix 안 거치고 명령으로 직접 처리**
어차피 prefix 없이 셸에서 바로 실행 가능:
```bash
tmux kill-window         # 현재 윈도우
tmux kill-session        # 현재 세션 전체
tmux kill-server         # tmux 통째로
tmux list-sessions       # 살아있는 세션 보기
tmux kill-session -t 이름  # 특정 세션
```

지금 상황(tmux 안인지/밖인지, prefix 설정) 알려주면 맞춰서 알려드릴게요.
