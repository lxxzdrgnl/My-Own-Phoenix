원인 정리됐어요. **Wayland + fcitx5에서 흔한 문제**입니다.

## 무슨 일이 일어나는지

1. Hangul/오른쪽 Alt 누름 → fcitx5가 Wayland compositor에 "IM 상태 바꿔" 요청 (왕복 통신)
2. **그 사이에 타이핑하면** 이전 IM(예: 영문) 상태로 처리됨 → 한영 전환이 "씹힌 것처럼" 보임
3. 게다가 `ShowInputMethodInformation=True`라 매번 "한국어/English" 팝업이 떠서 체감 지연 더 큼

## 즉시 적용할 수 있는 설정 변경

`~/.config/fcitx5/config` 에서 두 부분 수정 추천드려요:

```ini
[Behavior]
ShowInputMethodInformation=False        # True → False
ShowFirstInputMethodInformation=False   # 이미 False면 OK
ShareInputState=All                     # No → All (앱마다 상태 공유, 더 일관적)
```

그 다음 `fcitx5 -r` 또는 `pkill fcitx5 && fcitx5 -d` 로 재시작.

## 그래도 안 되면

Wayland 자체의 input-method 프로토콜 지연이라 근본 해결은 어려운데, 다음 중 하나 시도해보세요:

| 옵션 | 효과 |
|---|---|
| 트리거 키를 **오른쪽 Alt 하나만** 남기기 (Hangul 제거) | 일부 키보드/드라이버에서 Hangul 키 처리가 느림 |
| `EnumerateSkipFirst=True` | IM이 2개일 때 enumerate 로직 단순화 |
| X11 세션으로 로그인 | fcitx5는 X11에서 훨씬 빠름. GNOME에서 로그인 시 톱니바퀴로 "GNOME on Xorg" 선택 |

지금 바로 첫번째(`ShowInputMethodInformation=False` 등) 적용해드릴까요? 그게 80% 케이스에서 체감 차이 가장 큽니다.

### A. **kime 설치** (가장 추천 — 한국어 전용 IME)
