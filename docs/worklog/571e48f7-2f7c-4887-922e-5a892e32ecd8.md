## 포트포워딩 vs Tailscale

**포트포워딩의 단점들** (Tailscale 이 다 해결)

| 항목 | 포트포워딩 | Tailscale |
|---|---|---|
| **공유기 설정** | 모델마다 다른 메뉴 헤매기 | 필요 없음 |
| **공인 IP** | 필요 (없으면 DDNS) | 필요 없음 |
| **공유기 권한** | 본인 공유기만 가능 (회사/카페/호텔 X) | 어디서든 OK |
| **CGNAT 환경** | 통신사가 막아두면 불가능 | 우회됨 |
| **보안** | 22 포트 인터넷 노출 → 봇이 매일 두드림, fail2ban 등 별도 방어 필요 | 외부 노출 0, WireGuard 암호화 P2P |
| **IP 변동** | 공유기 재시작 시 DDNS 갱신 | 디바이스 이름으로 접속 (`desktop` 같은) |
| **여러 기기** | 기기마다 포트 다르게 (22, 2222...) | 각 디바이스가 자기 IP 가짐 |
| **방화벽 통과** | 막힌 환경(카페 와이파이) 안 됨 | UDP hole punching 으로 거의 다 됨 |
| **자동 키 관리** | SSH 키 / known_hosts 수동 | MagicDNS 로 자동 |

## 가장 큰 차이

**포트포워딩 = 집 현관문에 자물쇠 하나 더 다는 것** (문 자체는 길거리에 노출)

**Tailscale = 집을 사설 골목 안에 두는 것** (애초에 외부에서 안 보임)

특히 보안 면에서 차이가 큰데, 22 포트 인터넷 노출하면 *진짜* 매일 수천 번 brute-force 시도 들어옵니다 (`/var/log/auth.log` 보면 깜놀). Tailscale 은 외부에 포트 자체가 안 열려있어서 그게 0건.

## 추가 보너스

- **MagicDNS**: `ssh desktop` 만 쳐도 됨 (IP 외울 필요 X)
- **Taildrop**: 폰 ↔ 데스크탑 파일 전송
- **Exit node**: 외출 중 공용 와이파이에서 집 네트워크 통해 인터넷 (간이 VPN)


```bash
sudo systemctl status ssh
```

`active (running)` 나오면 OK. 안 깔려있으면:
```bash
sudo apt update && sudo apt install -y openssh-server
sudo systemctl enable --now ssh
```
